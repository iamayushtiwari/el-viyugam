# Pull Request Guidelines

Follow these rules before submitting a pull request.

---

## Requirements

- Keep pull requests focused on a single change  
- Link related issues whenever possible  
- Ensure code builds successfully before submitting  
- Explain what and why the change was made  

---

## Required Proof

- UI changes: include screenshots  
- Backend changes: include API or Postman response screenshots  
- Testing changes: include test result screenshots  

---

## Before Submitting

- Re-check code quality  
- Remove unused code or logs  
- Ensure proper formatting
