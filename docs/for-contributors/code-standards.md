# Code Standards

This document helps maintain clean, consistent, and easy-to-understand code across the project.

---

## General Guidelines

- Write clean and readable code  
- Keep functions small and focused on a single responsibility  
- Follow the existing project structure  

---

## Naming Conventions

- Use meaningful and descriptive names for variables and functions  
- Keep naming consistent across the project  
- Prefer clarity over short or unclear names  

---

## API Design

- Keep endpoints simple and consistent  
- Return clear and structured responses  
- Maintain predictable request and response formats  

---

## UI Guidelines

- Build reusable components wherever possible  
- Maintain consistent layout and design structure  
- Avoid duplicating UI logic  

---

## Code Comments

- Add comments only where the logic is not immediately clear  
- Keep comments simple and meaningful  
- Avoid over-commenting obvious code  

---

## Good Practices

- Remove unused or redundant code  
- Avoid hardcoded values when possible  
- Keep logic separated based on responsibility
